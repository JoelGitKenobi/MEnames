Elcor
Military vessel names

General rule:
Elcor naming must be source-pool driven, not stem-and-tail driven.
Do not invent elcor names by recombining tiny syllable sets.
Build conventions from attested full-name shapes, attested toponyms, attested ship and class names, and attested legendary figures.
Only corvettes use a serial-only system.

Attested source pools for extrapolation

Corvette
Format: JFV YYY-NNN
JFV = Judicial Fighting Vessel
YYY = last 3 digits of the construction year
NNN = 001-999 production order within that year
Rule: corvettes do not receive proper names. This is the only elcor military hull that is purely serial.
Examples: JFV 178-011, JFV 186-004.

Frigate
Format: JFV <DekuunaGeographicName>
DekuunaGeographicName = curated list of rivers and mountains on Dekuuna
Rule: frigates are named after rivers and mountains on Dekuuna. Do not use personal names, legends, or class names here.
Implementation note: if the geography list is incomplete, this category needs more source collection first. Do not fill it with synthetic syllable names.
Example: JFV Droriuntavilusium.

Destroyer
Format: JFV <HistoricServiceName>
HistoricServiceName pool = attested elcor soldiers, sailors, admirals, tacticians, explorers, and public servants with a service profile
Recommended short-name pool = {Hiwuraango, Harta, Orotan, Calyn, Kojet, Terojum}
Rule: destroyers use shorter honor names than cruisers. Prefer single names over compound names.
Examples: JFV Hiwuraango, JFV Harta, JFV Orotan.

Cruiser
Format: JFV <HistoricServiceName>
HistoricServiceName pool = same category as destroyers, but compound names are allowed and preferred when available
Rule: cruisers are named for famed elcor soldiers and sailors of the past. Use full attested names when the honoree is normally referenced that way.
Examples: JFV Moireturanum, JFV Toruun Thordaa, JFV Chirrano Xultonnos.

Battleship
Format: JFV <FableOrLegendName>
FableOrLegendName pool = {Denaususnomun, Oribosanti, Wonnumetenro, Or-Gyclyn, Ralmonisuuk, Raxuu}
Rule: battleships use elcor fable and legend figures, following the dreadnought convention from the Judicial Armada article.
Examples: JFV Denaususnomun, JFV Oribosanti.

Titan
Format: JFV <FableOrLegendName>
FableOrLegendName pool = same as battleship
Rule: titans use the same fable and legend pool as battleships. Reserve the most revered or oldest-sounding entries for the largest hulls.
Examples: JFV Wonnumetenro, JFV Ralmonisuuk.

Juggernaut
Format: JFV <FableOrLegendName>
FableOrLegendName pool = same as battleship
Rule: juggernauts do not need a separate naming grammar. Reuse the battleship and titan mythic pool instead of inventing a new format.
Examples: JFV Or-Gyclyn, JFV Raxuu.

Colossus
Format: JFV <FableOrLegendName>
FableOrLegendName pool = same as battleship
Rule: colossi use the same fable and legend pool as other elcor capital hulls. No extra serial or role code.
Examples: JFV Ralmonisuuk, JFV Denaususnomun.

Ship class / design names

General rule:
Elcor class designations should look like named ship classes, not machine catalog strings.
Attested examples are class-name style entries such as Orison Class and Taarishan-Class.

Shared format for all ship size and role combinations:
<ClassName>-class
Alternative style allowed: <ClassName> Class

ClassName pool:
Start with attested names {Orison, Taarishan}.
Expand using curated historical ship names or attested elcor proper names that already feel like vessel classes.

Rule:
Do not encode hull size, role, doctrine, mark number, or software package into the class name.
If Stellaris requires one design name per size-role slot, assign different names from one shared class-name pool rather than inventing separate naming grammars for every slot.
Do not use formats like GMB-CV-ANCHOR-Mk10 for elcor.

Examples:
Orison-class
Taarishan-class

Fleet names

Format: $ORD$ Judicial Flotilla
Rule: use one sequential convention for both fleet_names and military fleet names. The state naval arm is the Judicial Armada, so the default fleet label should reference that institution directly.
Example: 3rd Judicial Flotilla.

Army names

Format: $ORD$ Right Fist Detachment
Rule: army naming should reference the Right Fist of the Courts rather than generic weapon-platform language.
If differentiated army-type labels are needed, keep "Right Fist" fixed and vary only the last noun: Detachment, Cohort, Battery.
Example: 5th Right Fist Detachment.

Military fleet names

Use the same convention as fleet names.
Format: $ORD$ Judicial Flotilla
Example: 2nd Judicial Flotilla.

Civilian vessel names

Science
Format: MSV <ProperShipName>
ProperShipName pool = civilian ship names and neutral elcor proper names suitable for ships
Attested anchor = Davrona
Rule: civilian science ships use a proper ship name with MSV prefix. No serial suffixes.
Example: MSV Davrona.

Constructor
Format: MSV <ProperShipName>
ProperShipName pool = same category as science ships, with preference for trade, settlement, or historical-vessel names
Rule: constructor ships use the same civilian naming grammar as science vessels. Do not invent a separate synthetic world-name generator just for builders.
Example: MSV Davrona.

Troop Transport
Format: JFV <ProperTransportName>
ProperTransportName pool = named military vessel pool, with preference for transport, carrier, and mobilization-associated names
Rule: elcor troop transport is a military specialty, so use JFV rather than MSV. If a separate transport-name pool does not exist yet, share the cruiser and carrier proper-name pool instead of creating a serial format.
Example: JFV Moireturanum.

Planets

Use one generic convention only.

Format: full toponym from a curated elcor place-name pool
Toponym pool = {Dekuuna, Ekuna, Oltan, Sangel, Thunawanuro, Seruun, Malvuon}
Rule: do not derive planet names from personal-name stems. New planet names must be added as whole place names, modeled on attested elcor worlds, moons, and settlements.
No climate subclasses.
Examples: Dekuuna, Oltan, Seruun, Malvuon.
