Naming conventions prompt template:
Let's create conventions for a new namelist in stellaris for the X empire. The conventions need to be strict and follow clear definitions.
The definitions should either use language and sound patterns from the empires lore culture, or examples of existing names. Usage of existing naming conventions is prefered, but where none such exist, we will need to extrapolate.
A naming convention can't be: "grand words, mythological references, or historical figures". It needs to be more specific and follow a clear pattern.
An example of a clear pattern is: "Use different combinations of the following syllables: 'ka', 'ra', 'ta', 'ma', 'la' for corvettes, and add a number at the end to indicate the ship's production order (e.g., 'Kara-1', 'Rata-2')." or "Known earth cities" "Historical battles" etc.
We need one naming convention per category, presented in a listed order. We are allowed to use sequential numbering in most names and for categories where there is a lot of entities to name (e.g ships and fleets).
There are sometimes real conventions that exist withinthe lore of mass effect, these should be used where possible but scaled and fitted to the actual categories requred.

Character names
General format: FaceName (SoulFull)
FaceName uses the FaceName generator.
SoulFull uses the full SoulName generator.

FaceName generator:
Prefix P ∈ {Del, Zym, Yss, Kho, Opo, Kah, Bel, Vas, Nya}
Optional connector C ∈ {a, e, i, o, an, yn}
Suffix S ∈ {dis, ynder, lai, old, es, je, an, is}
Rule: FaceName = P + (C) + S

SoulName generator:
Verb V ∈ {Regards, Illuminates, Interprets, Catalogues, Measures}
Object O ∈ {Works, Gift, Speech, Silence, Currents}
Referent R ∈ {Enkindlers, Language, Kahje, Shallows}
Mood M ∈ {in Despair, in Patience, in Wonder}
Rule: SoulFull = V + " the " + O + " of the " + R + " " + M

Example: Zymandis (Regards the Works of the Enkindlers in Despair)
